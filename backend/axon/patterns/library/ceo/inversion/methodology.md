# Inversion

Inversion flips a problem on its head. Instead of asking "How do we build a great company?" you ask "What would guarantee we destroy this company?" By mapping the paths to failure explicitly, you often uncover blind spots that positive-framing brainstorming misses entirely.

Use inversion when you are planning strategy, evaluating risk, or sense that optimism bias is clouding the room. It is especially powerful during annual planning, pre-mortems, and when evaluating whether to enter a new market.

**Steps:** (1) State your goal. (2) Reframe it: "What actions would reliably produce the opposite outcome?" (3) List every failure mode you can imagine — losing key talent, ignoring customers, shipping too slowly, burning cash. (4) For each failure mode, check whether you are currently doing any of those things. (5) Build safeguards or stop doing the harmful behaviors.

A common mistake is treating inversion as a pessimism exercise and demoralizing the team. Frame it as a tool for clarity, not doom. Another pitfall is generating the failure list but never mapping it back to current behavior — the value is in the comparison, not the list itself.
