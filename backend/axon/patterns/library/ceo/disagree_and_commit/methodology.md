# Disagree and Commit

This pattern separates the debate phase from the execution phase. During debate, everyone is expected to voice genuine concerns and push back hard. Once a decision is made, everyone — including dissenters — commits fully as if it were their own idea. This prevents both false consensus and passive sabotage.

Apply this when the team is split on a direction and waiting for unanimity would cost critical time. It is especially useful for two-way door decisions where the cost of delay exceeds the cost of a suboptimal choice.

**Steps:** (1) Set a clear decision deadline. (2) Ensure all perspectives are heard — specifically ask dissenters to articulate their concerns. (3) The decision owner makes the call and states it explicitly. (4) Everyone commits to making the chosen path succeed. (5) Set a review checkpoint to revisit if evidence changes.

The biggest mistake is "disagree and sabotage" — paying lip service to commitment while undermining execution. The second mistake is skipping the disagree phase, which produces decisions that lack stress-testing and buy-in.
