# Progressive Disclosure

Progressive disclosure is the practice of showing only the essential information and controls at first, then revealing additional complexity as the user demonstrates readiness or need. It makes simple tasks simple while still supporting advanced workflows. The interface grows with the user's understanding.

Apply this when designing features with both novice and expert users, when a form has many optional fields, or when a screen risks overwhelming first-time users. It is a core technique for making powerful tools feel approachable.

**Steps to apply:**
1. Separate the interface into primary actions (used by 80% of users) and secondary actions (used by 20%).
2. Present primary actions prominently and immediately.
3. Tuck secondary actions behind progressive triggers — expandable sections, "Advanced" links, hover states, or contextual menus.
4. Ensure discoverability — the path to advanced features must be visible even if the features themselves are not.

**Common mistakes:** Hiding important features so deeply that even power users cannot find them. Using progressive disclosure as an excuse to avoid simplifying — the primary layer must still be well-designed. Revealing too many layers at once, defeating the purpose.
