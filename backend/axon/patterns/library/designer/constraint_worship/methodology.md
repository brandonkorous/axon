# Constraint Worship

Constraint worship is the practice of deliberately imposing limits to force creative clarity. Instead of asking "what should we add?" ask "if we could only have three elements on this screen, which three?" Constraints eliminate compromise-by-addition, where every stakeholder gets their feature and the result serves no one well. The best design decisions emerge under tight constraints.

Apply this when a design is growing bloated, when stakeholders cannot agree on priorities, or at the start of a project to establish focus. Artificial constraints are especially useful when real ones are absent.

**Steps to apply:**
1. Name the constraint explicitly — "only 3 features," "must fit on a single screen," "one-sentence description."
2. Force-rank everything against the constraint. What survives?
3. Design for only the survivors. Set everything else aside entirely.
4. Evaluate the constrained version. Often it is better than the unconstrained original.
5. Relax the constraint slightly only if testing reveals a critical gap.

**Common mistakes:** Treating constraints as problems to work around instead of tools to work with. Setting fake constraints then ignoring them under pressure. Using constraints to avoid hard design work rather than to focus it.
