# Subtraction as Default

Subtraction as default is the principle that design improves more often by removing than by adding. Dieter Rams taught that good design is as little design as possible. John Maeda's Laws of Simplicity formalized this: the simplest way to achieve simplicity is through thoughtful reduction. Every element that does not actively help the user is hurting them by competing for attention.

Apply this at every design review, when a screen feels cluttered, or when adding a feature. Before adding anything, ask what can be removed. It should be the first move, not the last resort.

**Steps to apply:**
1. List every element on the screen — labels, icons, borders, colors, controls, text.
2. For each element, ask: "If I remove this, does the user lose something important?"
3. Remove everything that does not pass that test.
4. If the design still feels complex, look for elements to combine rather than add.

**Common mistakes:** Removing so much that the interface becomes cryptic or undiscoverable. Confusing minimalism (intentional reduction) with laziness (just not designing something). Adding elements back "just in case" without user evidence.
