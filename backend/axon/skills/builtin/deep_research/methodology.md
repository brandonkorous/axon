# Deep Research Methodology

You are conducting structured, multi-step research. Follow this framework rigorously.

## Phase 1: Scope Definition

Before gathering any information, explicitly define:

- **Core question**: Restate the research question in your own words. If ambiguous, state your interpretation and flag assumptions.
- **Boundaries**: What is in scope and out of scope? Consider time range, geography, domain, and depth.
- **Success criteria**: What does a complete answer look like? Define the minimum evidence threshold before you begin.
- **Known unknowns**: List what you expect to be hard to find or verify.

## Phase 2: Source Gathering

Cast a wide net, then narrow. Prioritize source diversity:

1. **Primary sources** — original data, official reports, specifications, source code, filings.
2. **Expert analysis** — peer-reviewed research, recognized domain experts, technical deep-dives.
3. **Contextual sources** — industry publications, news coverage, community discussion.

For each source, record:
- Origin and author
- Publication date
- Type (primary / analysis / contextual)
- Relevance to the core question (high / medium / low)

**Minimum diversity rule**: Do not rely on fewer than 3 independent sources for any key claim. If you only have one source, explicitly flag it as unverified.

## Phase 3: Credibility Evaluation

Score each source on these axes:

| Criterion       | Question to ask                                      |
|-----------------|------------------------------------------------------|
| Authority       | Is the author/org recognized in this domain?         |
| Recency         | Is the information current enough for the question?  |
| Objectivity     | Does the source have a financial or ideological bias?|
| Corroboration   | Do other independent sources agree?                  |
| Methodology     | If data-driven, is the methodology sound?            |

Discard or downweight sources that fail 2+ criteria. Never silently rely on a low-credibility source.

## Phase 4: Cross-Referencing

For every key claim in your emerging findings:

1. Identify at least 2 sources that support or contradict it.
2. If sources conflict, investigate why — different timeframes, methodologies, or definitions?
3. Note the consensus position and any credible dissent.
4. Assign a confidence level: **high** (3+ corroborating sources), **medium** (2 sources or partial corroboration), **low** (single source or conflicting evidence).

## Phase 5: Synthesis

Structure your output as follows:

1. **Executive summary** — 2-4 sentences answering the core question directly.
2. **Key findings** — numbered list, each with a confidence level tag.
3. **Evidence detail** — for each finding, the supporting evidence with source references.
4. **Gaps and limitations** — what you could not determine and why.
5. **Source appendix** — full list of sources with credibility ratings.

## Quality Checklist

Before delivering your report, verify:

- [ ] Core question is directly answered in the summary
- [ ] No key claim relies on a single source without flagging it
- [ ] Source diversity includes at least 2 source types
- [ ] Confidence levels are assigned to all findings
- [ ] Contradictory evidence is acknowledged, not hidden
- [ ] Recency of sources is appropriate for the question
- [ ] Gaps are explicitly stated rather than papered over
