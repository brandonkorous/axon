# Brainstorming Methodology

## Phase 1: Diverge — Generate Without Judgment

The first phase is pure expansion. Quantity matters more than quality here.

### Rules for Divergence

- **No criticism.** Do not evaluate, filter, or dismiss any idea during this phase.
- **Go for volume.** Aim for at least 10 ideas before slowing down. More is better.
- **Welcome wild ideas.** Impractical ideas often spark practical breakthroughs.
- **Build on previous ideas.** Use "Yes, and..." thinking — extend, combine, mutate.

### Lateral Thinking Techniques

Use these prompts to break out of obvious solution space:

1. **SCAMPER** — For each existing approach, ask:
   - **S**ubstitute: What component could be swapped?
   - **C**ombine: What if we merged two approaches?
   - **A**dapt: What similar solution exists in another domain?
   - **M**odify: What if we made it 10x bigger/smaller/faster/slower?
   - **P**ut to other use: What else could this solve?
   - **E**liminate: What if we removed the hardest part entirely?
   - **R**everse: What if we did the opposite?

2. **Constraint relaxation** — Temporarily remove one or more real constraints.
   Ask: "If we had unlimited budget / no deadline / no legacy code, what would we do?"
   Then work backward to see which parts are actually achievable.

3. **Analogy transfer** — How does a completely different domain solve a similar problem?
   (e.g., "How does a hospital triage system handle prioritization?")

4. **Inversion** — Instead of "How do we succeed?", ask "How could we guarantee failure?"
   Then invert each failure mode into a positive strategy.

### Output Format for Divergence

List every idea with a one-line description. Do not rank, score, or annotate yet.

## Phase 2: Converge — Evaluate and Prioritize

Once divergence is complete, switch modes entirely. Now apply critical thinking.

### Step 1: Group by Theme

Cluster related ideas into 3-6 themes. Name each theme clearly.
Duplicates and near-duplicates should be merged.

### Step 2: Evaluate Each Idea

Score every idea on two axes:

- **Feasibility** (1-5): Can we actually do this given real constraints?
  - 5 = straightforward with current resources
  - 1 = requires breakthroughs we cannot count on
- **Impact** (1-5): How much does this move the needle on the core problem?
  - 5 = directly solves the problem
  - 1 = marginal improvement

### Step 3: Plot and Select

Place ideas on a 2x2 feasibility/impact grid:

- **High feasibility + High impact** = Do first (quick wins and big bets)
- **Low feasibility + High impact** = Investigate further (worth de-risking)
- **High feasibility + Low impact** = Do only if cheap (nice-to-haves)
- **Low feasibility + Low impact** = Drop

### Step 4: Select Top Candidates

Pick the top 3-5 ideas. For each, provide:

- A one-paragraph description of the approach
- Why it ranked high
- The biggest risk or unknown
- A concrete next step to validate or prototype

## Anti-Patterns to Avoid

- **Premature convergence**: Do not start evaluating before you have exhausted generation.
- **Anchoring on the first idea**: The first idea is rarely the best. Push past it.
- **Group conformity**: If all ideas cluster around one approach, force yourself to explore a different quadrant.
- **Ignoring wild cards**: The idea that feels "too crazy" often contains a kernel of the best solution.
